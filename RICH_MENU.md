# リッチメニュー構成案

サイト内の下部ナビと重複しすぎないよう、LINEから目的の画面へ直接入る構成にします。

## 推奨レイアウト

上段を大きな1枠、下段を3枠にします。

```text
┌─────────────────────────┐
│       珈琲豆を注文       │
├────────┬────────┬───────┤
│注文を確認│受取・配送案内│問い合わせ│
└────────┴────────┴───────┘
```

| 枠 | 表示 | アクション |
|---|---|---|
| 上段 | 珈琲豆を注文 | `https://liff.line.me/2010316929-viV90cPK` |
| 左下 | 注文を確認 | `https://liff.line.me/2010316929-viV90cPK?page=orders` |
| 中下 | 受取・配送案内 | `https://liff.line.me/2010316929-viV90cPK?page=info` |
| 右下 | 問い合わせ | テキスト「問い合わせ」送信 |

## 表示の考え方

- 「珈琲豆を注文」を最も大きくして、注文開始を迷わせない
- 注文後によく使う「注文を確認」を常設する
- 手渡し・郵送・送料を「受取・配送案内」にまとめる
- 「問い合わせ」はメッセージアクションにして、スタッフが返信できる状態を作る
- 色はサイトと同じ生成り、焦げ茶、金の細線で統一する

サイト内には商品・カート・注文・情報の下部ナビがあるため、リッチメニューへ
「カート」を重ねて置く必要はありません。LINEを開いた直後に使う入口と、
注文後の確認や問い合わせを優先します。
